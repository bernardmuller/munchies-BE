MUNCHIES API

version 0.2

Routes

Authentication routes
  • post /register - requires email and password
  • post /login - requires email and password, sets LOGGED_IN_USER cookie => returns JWT token and user ID
  • get /auth - decode JWT => returns object with auth
  • post /logout - clears all cookies
menus routes
  • get /menus - requires JWT => returns list of all menus
  • get /menus/:id - requires JWT => returns single menu object, showing details and list of meals
  • put /menus/:id - requires JWT => updated menu with req.body
  • post /menus - requires JWT => creates new menu with bland details and "Untitled Menu" as name
  • delete /menus/:d - requires JWT => deletes meal
meals routes
  • get /meals - requires JWT => returns list of all meals
  • get /meals/:id - requires JWT => returns meal object
  • put /meals/:id/edit - requires JWT => updated meal with req.body
  • post /meals/create - requires JWT => creates new meal with blank details and "Untitled Meal" as name
  • delete /meals/:mealID - requires JWT => deletes meal
ingredients routes
  • get /ingredients - requires JWT => returns list of all meals
  • get /ingredients/:id - requires JWT => returns singe ingredient object
  • put /ingredients/:id - requires JWT => updated ingredient with req.body
  • post /ingredients - requires JWT => creates new ingredient with req.body.name
  • delete /ingredients/:id - requires JWT => deletes meal
